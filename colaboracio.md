# Treball col·laboratiu amb GitHub

Aquest document explica tres conceptes essencials del flux de treball a GitHub: **forks**, **pull requests (PRs)** i **issues**, a més d’una reflexió sobre els avantatges de treballar en equip amb aquestes eines.

---

## Què és un *Fork*?

Un **fork** és una còpia d’un repositori d’un altre usuari al teu propi compte de GitHub.  
Serveix per poder experimentar o fer canvis sense afectar el projecte original.

---

## Què és una *Pull Request* (o *Merge Request*)?

Una ***pull request*** és una petició per demanar que els teus canvis (fets en el teu *fork* o branca) s’incorporin al projecte original.  
Permet revisar, comentar i aprovar els canvis abans de fusionar-los. Alguns proveidors de repositoris l'anomenen, més encertadament, ***Merge Request***.

---

## Què és una *Issue*?

Una **issue** és un mètode per comunicar problemes, idees o propostes de millora dins d’un projecte.  
Les issues ajuden a fer seguiment de tasques o errors d’una manera organitzada.

---

## Reflexió final

Treballar amb el flux de GitHub (forks, PRs i issues) aporta molts **avantatges**:

- Facilita la **col·laboració** sense trepitjar el codi dels altres.  
- Promou la **revisió de codi**, que millora la qualitat del projecte.  
- Garanteix un **registre clar** de discussions i decisions.  
- Permet que cada membre aprengui dels altres i mantingui un flux de treball ordenat.

> En resum: GitHub no només és una eina per guardar codi, sinó també una **plataforma de col·laboració i aprenentatge**.
